# Онлайн Нотатник — Командний Проєкт (з дисципліни «Інформаційні технології»)

Проєкт «Онлайн Нотатник» — це веб-застосунок, створений командою студентів КНУ в рамках лабораторної роботи №2. Метою роботи було розробити простий, зручний та функціональний інструмент для управління особистими нотатками, який водночас демонструє повний цикл розробки програмного забезпечення: від планування та архітектури до реалізації, тестування й документування. Розробка велася за гнучкою методологією Agile/Kanban із веденням задач у Jira.

## Мета та призначення системи

Система «Онлайн Нотатник» створена як інтуїтивно зрозумілий сервіс для швидкого збереження, редагування та організації текстової інформації. На відміну від складних продуктів (Notion, Evernote), наш застосунок орієнтований на студентів та невеликі команди, яким потрібен легкий та зрозумілий інструмент. Основна мета полягала у створенні MVP з можливістю подальшого масштабування.

## Функціональні можливості

Авторизація та безпека:
- Реєстрація користувача (Email + Password)
- Вхід у систему
- Хешування паролів

Робота з нотатками:
- Створення нотаток
- Редагування тексту
- Перегляд нотаток на Dashboard
- Видалення з перенесенням у «Кошик»
- Відновлення нотаток
- Додавання у «Вибране»
- Сортування нотаток

Спільний доступ:
- Генерація публічних посилань
- Режими доступу: приватна або публічна нотатка

Робота з файлами:
- Прикріплення файлів до нотаток
- Завантаження прикріплених файлів
- Збереження даних про файли в базі даних

## Технічна архітектура

Архітектура застосунку побудована за трирівневою моделлю (3-Tier Architecture), що забезпечує розділення логіки, гнучкість та масштабованість системи:

1. Frontend (Client) — SPA застосунок, що відповідає за інтерфейс користувача.
2. Backend (Server) — REST API, що реалізує бізнес-логіку.
3. Database (Storage) — реляційна база даних MySQL.

Інструменти:
- Jira для планування та управління беклогом
- GitHub для командної роботи та контролю версій

## Структура бази даних

База даних містить наступні таблиці:
- Users — облікові записи користувачів
- Notes — нотатки, прив’язані до користувача
- Files — прикріплені файли
- Permissions — правила доступу для спільного користування нотатками

Основний зв’язок: Users (1) — Notes (N).

Backend команда:
- Реалізація REST API
- CRUD-операції

- Тестування UI
- Звітність про знайдені помилки

## Методологія розробки

Розробка проводилася за методологією Agile/Kanban. Усі задачі створювалися, пріоритезувалися та відслідковувалися в Jira. Ітерації включали створення MVP, реалізацію основних функціональних модулів (Sharing, File Upload), а також опрацювання можливих розширень (коментарі, покращений текстовий редактор). Кожна задача проходила шлях від беклогу до статусу «Done».

Система має чітку архітектуру, дотримується принципів багаторівневої побудови та готова до подальшого масштабування.
